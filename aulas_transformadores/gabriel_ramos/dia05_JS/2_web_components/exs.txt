1) Essencialmente, o que são Web Components?

RESPOSTA: É uma série de especificações que visa tornar componentes na web reutilizáveis independente de frameworks.

2) Você precisa de algum framework/biblioteca para aplicar componentização em uma interface? Por quê?

RESPOSTA: Não. Componentizar é basicamente o esforço de definir limites claros para componentes de interface, 
isso quer dizer, separar as responsabilidades de cada bloco dentro de uma aplicação front-end.
